Glossary


An entity that has a state and behavior associated with it.

A collection of objects. A class contains the blueprints of objects that are created. It is a logical entity that contains some attributes and methods.

The creation and naming of an object. It's an instance of a class.

A collection of currently defined symbolic names along with information about the object that each name references.

A special kind of method that is used when instantiating an object using the definitions found in a class.

Provides a defined, reliable and consistent context for the tests. Fixtures define the steps and data that constitute the arrange phase of a test.

Hint a data type for a variable. A way to make Python feel statically typed and can help structure your projects. Also, type hints can help to debug and prevent some dumb mistakes.

A way to convert the variable data type into another data type in order to do the operation required to be performed by users.

Testing individual components of a software program or application. It's used to check that all the individual parts are working as intended.

The particular member belongs to a type itself, rather than to an instance of that type, and its lifetime is the entire program.

Methods which require an object of its class to be created before it can be called. The most common type of methods in Python classes. To invoke an instance method, we have to create an Object of the class in which the method is defined.

A typical property declared in a class is an instance property, meaning that you have a copy of that property’s value for each instance of the class.

Methods that can be called without creating an object of a class. They are referenced by the class name itself or reference to the object of that class.

Properties that have a single value for the entire class, regardless of the number of instances of the class that exist.

Putting data and methods for that data in one unit, putting restrictions on the data and its methods. A class is an example of encapsulation as it encapsulates all the data that is member functions, variables, etc.

The capability of one class to derive or inherit the properties from another class. It's just like real-world relationships, you can reuse the code, and allows you to add more features to a class without modifying it.

Polymorphism means having many forms. The same function name is used for different methods, but each with its own signature.

Abstraction is like creating a blueprint of something. It allows you to declare a set of abstract methods without any implementation that must be created within any child classes built from the abstract class.

SOLID is an acronym for the first five object-oriented design principles by Robert C. Martin.
Single-Responsibility - One class should only do one thing.
Open-Closed - Objects or entities should be open for extension but closed for modification. This means you can extend the class but not modify it.
Liskov Substitution - Objects of a superclass should be replaceable with objects of its subclasses without breaking the application.
Interface Segregation - A client should not be exposed to methods it doesn't need. Declaring methods in an interface that the client doesn't need pollutes the interface and leads to a “bulky” or “fat” interface.
Dependency Inversion - High level modules should not depend on low level modules; in fact both should depend on abstractions.

A general repeatable solution to a commonly occurring problem in software design. It is a description or template for how to solve a problem that can be used in many situations.
