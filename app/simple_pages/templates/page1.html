Git


What are Branches?

Branches are lightweight movable pointers to one of the commits. Think of branches like they are separate environments. When you make changes and commit them to a branch the pointer of that branch moves to the latest commit. You start off with a default branch called main or master. Then you can diverge to another branch to make changes without affecting the original branch. Then you can merge back to main branch.


What is a Merge?

Merge is combining the forked branch back to the original branch. For instance, you work on a feature in a separate branch, and once it completed and tested you can combine the feature back to the main branch with the main program. It will basically add the new code to the main branch.


What is a Commit?

Commits are snapshots of the project's current staged changes. A commit basically means that you are saving changes of the project to the computer/local repository.


What are Tags?

Tags are used to identify specific points/commits in history of the project. They are generally used to mark a version of the project that was released, for example, v3.04


What are Repositories?

Repository is just a project. It's just a collection of files of different versions of a project. Also, it tracks all the changes made to all the files in the project to build a history overtime.
