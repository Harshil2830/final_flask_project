Docker


What is a Docker Container and an Image?

A Dockerfile is a text document that contains commands to assemble a docker image. A Docker Image is a template or a blueprint used to build a docker container. A Docker Container is a running instance of a docker image, and contains all the necessary packages to run an application.


How to get started

  1. Create a Dockerfile using any text editor.
  2. In the terminal use docker build . to build and image from the Dockerfile
  3. In the terminal use docker images to find image id
  4. In the terminal use docker run -it -p port_number -d image_id to create a container based on the image and get access to it through the terminal

Some useful docker commands

  • docker build - used to build a docker image from a Dockerfile
  • docker run - used to create an instance of a docker image
  • docker compose up --build - used to build, (re)create, start, and attach to containers for a service
  • docker images - used to list most recently created images
  • docker ps - used to list running containers
  • docker kill - used to kill one or more running containers
  • docker rm - used to remove containers
  • docker pull/push - used to pull or push an image or a repository
