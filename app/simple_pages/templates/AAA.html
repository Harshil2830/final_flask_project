AAA Testing


What is Unit Testing?

Testing individual components of a software program or application. It's used to check that all the individual parts are working as intended. Testing enables you to see what the software does and how well it does it so that the business can measure the quality of the software before it goes live. The benefits of testing include preventing bugs, reducing development costs and improving performance.


Arrange-Act-Assert

AAA is a powerful pattern for writing well-structured tests.

Arrange - inputs and targets. This phase is where you set up the test cases, like creating an object, or connecting to a database.
Example: tuple_list = (1, 2) is setting up the tuple input parameter.

Act - on the target behavior. This phase is where you perform the action on the thing being tested, like calling a function or API.
Example: calculation = Addition.create(tuple_list) is where the create function is called using the tuple_list input.

Assert - expected outcomes. This phase is where you verify the outcome/response from the previous action. This is where its determined whether the test passes or fails.
Example: assert calculation.get_result() == 3 is where you test the outcome, and whether the addition passes or fails.
